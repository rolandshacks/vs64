
# VS64 Manual Configuration []()

Most of the times, there is no need to, but VS64 supports deep customization
of the toolchain, arguments being used during the build, as well as the
integration of emulators for debugging sessions.

Things that can be configured if needed:

- Assembler and compiler install directories
- Additional global build arguments, defines and includes
- Overwriting of internally used tools like Ninja, Python or JRE
- Customization of the integrated BASIC compiler and resource compiler
- Adjustments of emulator executable paths or arguments
- Various flags like logging level, url of tool bundles, default charset
